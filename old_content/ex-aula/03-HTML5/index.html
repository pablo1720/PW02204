Guía de HTML5

Miércoles 29 de agosto de 2018

¿Qué es HTML5?

HTML5 es la última versión de html. Esta nueva versión junto con CSS3, define los nuevos estándares del desarrollo web. No solo se limita a incorporar nuevas etiquetas o atributos, si no que muchas características nuevas y proveer una plataforma compleja de desarrollo para aplicaciones web (mediante APIs).

Esta nueva version, no solo está designada para sustituir HTML4, si no que también XHTML 1 y DOM nivel 2. Nos permite una mayor interacción entre las paginas web y el contenido multimedia (video, audio, entre otros), así como menos complejidad a la hora de codificar nuestro diseño básico.

Algunas de las nuevas características de HTML5 son:

  • Semántica: Nuevas etiquetas semánticas para estructurar los documentos HTML. Permite describir con mayor precisión cual es el contenido de la página web.
  • Conectividad: Permite comunicarse con un servidor de maneras nuevas e innovadoras.
  • Sin conexión y almacenamiento: Permite a las páginas web almacenar datos localmente en el lado del cliente y poder operar sin conexión de una forma más eficiente.
  • Multimedia: Nuevos elementos multimedia. Nos otorga un excelente soporte para utilizar contenido multimedia como lo son audio y video nativamente.
  • Gráficos y efectos 2D/3D: Integración de gráficos vectoriales. Proporciona una amplia gama de nuevas características que se ocupan de los gráficos en la web como lo pueden ser canvas 2D, WebGL, SVG (como sustitución de <object> ), entre otros.
  • Rendimiento e integración: Proporciona una mayor optimización de la velocidad y un mejor uso del hardware.
  • Acceso al dispositivo: Proporciona APIs para el uso de varios componentes internos de entrada y salida de nuestro dispositivo.
  • CSS3: Nos ofrece una nueva gran variedad para hacer diseños más sofisticados.

Semántica

Una de las novedades que se mencionó anteriormente son las etiquetas nuevas que se incorporan en HTML5 que ayudan a definir la estructura del documento y permiten que las páginas web sean mejor indexadas por los buscadores.

Una etiqueta se califica como semántica si tiene que ver con el significado, es decir, si nos informa sobre lo que trata su contenido. Por ejemplo, la etiqueta <section> nos dice que contiene una sección o un capítulo dentro de la página.
Estas nuesvas etiquetas se pueden clasificar en dos grupos:

  • Etiquetas que extienden a las actuales, como <video>, <audio> o <canvas>, y que además agragan nuevas funcionalidades a los documentos HTML (que se pueden controlar desde javascript).
  • Etiquetas que componen la web semántica, es decir, que no incluyen nuevas funcionalidades pero sirven para estructurar sitios web, y añadir un significado concreto, más allá de las etiquetas generales como <div>.

Cabecera del documento

Además de las nuevas etiquetas introducidas por HTML5, el nuevo estándar propone pequeñas mejoras que podemos aplicar en la definición de los documentos, en concreto en la cabecera de los mismos.

DOCTYPE

El estándar XHTML deriva de XML, por lo que comparten muchas de sus normas y sintaxis. Uno de los conceptos fundamentales de XML es la utilización del DTD o Document Type Definition. El éstandar XHTML define el DTD que deben seguir las páginas y documentos XHTML. En este se definen las etiquetas que se pueden utilizar, atributos de cada etiqueta y el tipo de valores que puede tener cada atributo. 

 <!DOCTYPE html 
                PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN"
                "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

Esta es una de las 15 formas de declaraciones posibles en los estándares HTML4 y XHTML. En HTML5 se reduce la definición del tipo de documento a una sola posibilidad, por lo que no hay que preocuparse de elegir el tipo de documento correcto: 

<!DOCTYPE html>

Elemento raíz html

En todo documento HTML, su elemento raíz siempre es la etiqueta <html>.
Hasta antes de HTML5, este elemento raíz se definía de la siguiente manera: 

<html xmlns=http://www.w3.org/1999/xhtml
                lang="en"
                xml:lang="en">

No hay ningún problema en mantener esta sintaxis. Si se desea se puede mantener, ya que esto es valido en HTML5. Sin embargo alguna de sus partes ya no son necesarias, por lo que se pueden obviar.

El primer elemento del que podemos prescindir es el atributo xmlns. Se trata de una herencia de XHTML 1.0, que dice que los elementos de esta página están en el espacio de nombre XHTML, http://www.w3.org/1999/xhtml. Sin embargo, los elementos de HTML5 están siempre en este espacio de nombres, por lo que ya no es necesario declararlo explícitamente. Eliminar el atributo xmlns nos deja el elemento de la siguiente manera: 

<html lang="es" xml:lang="en">

En este caso ocurre lo mismo con el atributo xml:lang, es una herencia de XHTML que podemos eliminar, quedando finalmente la etiqueta de la siguiente manera: 

<html lang="es">

Elemento HEAD

El primer hijo del elemento raíz es generalmente el elemento <head>. El elemento <head> contiene los metadatos que proporcionan información extra sobre alguna página, como su título, descripción, autor, entre otros. Además, se pueden incluir referencias externas a contenidos necesarios para que el documento se comporte y se muestre de una manera correcta, como hojas de estilo (CSS) o scripts (JS). Este elemento ha obtenido pequeñas variaciones, pero que se deben de tener en cuenta. 

                <head>
                    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
                    <title>My Weblog</title>
                    <link rel="stylesheet" type="text/css" href="style-original.css" />
                    <link rel="alternate" type="application/atom+xml" title="My Weblog feed" href="/feed/" />
                    <link rel="search" type="application/opensearchdescription+xml" title="My Weblog search" href="opensearch.xml"  />
                    <link rel="shortcut icon" href="/favicon.ico" />
                </head>

Codificación del contenido

Cuando se piensa en "texto", probablemente sea una definición de "caracteres y símbolos que se ven en la pantalla del ordenador", pero realmente se tratan de bits y bytes. Cada cadena de caracteres que se muestra en la pantalla, se almacen en una codificación de caracteres en particular. Hay cientos de codificaciones de caracteres diferentes, algunas hechas u optimizadas para ciertos idiomas, como el ruso, chino o inglés, y otros que se pueden usar para múltiples idiomas. En general, la codificación de caracteres provee una correspondencia entre lo que se muestra en la pantalla y lo que un equipo realmente almacena en memoria.

Se puede pensar en la codificación de caracteres como una especie de clave de descifrado de texto. Cuando se accede a una secuencia de bytes, y se decide que es "texto", lo que se necestia saber es, qué codificación de caracteres se utiliza para que pueda decodificar los bytes en caracteres y mostrarlos de manera correcta.

Lo ideal es establecer esta codificación en el servidor, indicando el tipo en las cabeceras de respuesta: 

Content-Type: text/html; charset="utf-8"

No siempre se puede tener control sobre la configuración de un servidor HTTP. Por ejemplo, en la plataforma Blogger, el contenido es proporcionado por personas, pero los servidores son administrados por Google, por lo que estamos supeditados a su configuración.

El encabezado HTTP es el método principal, y anula la etiqueta <meta> si está presente. Pero no todo el mundo puede establecer encabezados HTTP, por lo que la etiqueta todavía tiene sentido. En el caso de HTML5, es aún más sencillo definir esta etiqueta meta:

<meta charset="utf-8">
A comparación de HTML4: 
<meta http-equiv="Content-Type" content="text/html; charset=UTF-8">

Dentro del elemento <head>, las etiquetas <link> son una manera de acceder o declarar contenido externo al documento actual, que puede cumplir distintos objetivos:

  • Enlaces a recursos externos que se van a utilizar para mejorar el documento actual.
  • Enlaces de hipervínculos que son enlaces a otros documentos.

El tipo de relación más utilizado es el siguiente: 

 <link rel="stylesheet" href="style-original.css" type="text/css" />

Etiquetas semánticas

Este tipo de etiquetas que componen la web semántica sirven para que cualquier mecanismo automático (navegador, motor de búsqueda, lector de feeds...) que lea un sitio web sepa con precisión qué partes de su contenido corresponden a cada una de las partes típicas de un sitio. Observando esas etiquetas semánticas estructurales, cualquier sistema podrá procesar la página y saber cómo está estructurada. A continuación algunas de estas etiquetas que introduce HTML5 en este apartado.

  • <section></section>: se utiliza para representar una sección "general" dentro de un documento, como un capítulo de un libro. Puede contener "subsecciones" y si lo acompañamos de etiquetas de encabezado ( h1-h6 ) se puede lograr una mejor estructura de toda la página creando jerarquías del contenido, algo muy favorable para el buen posicionamiento web.
  • <article></article>: representa un parte de una página que consiste en una composición autónoma en un documento, página o sitio web con la intención de que pueda ser reutilizado y repetido. Podría utilizarse en los artículos de los foros, una revista o el artículo de periódico, una entrada de un blog o un comentario escrito por un usuario.
  • <aside></aside>: representa una sección de la página que abarca un contenido relacionado con el contenido que lo rodea, por lo que se le puede considerar un contenido independiente. Este elemento puede utilizarse para efectos tipográficos, barras laterales, elementos publicitarios, para grupos de elementos de la navegación, u otro contenido que se considere separado del contenido principal de la página.
  • <header></header>: representa un grupo de artículos introductorios o de navegación. Está destinado a contener por lo general la cabecera de la sección (un elemento h1-h6 o un elemento hgroup), pero no es necesario.
  • <nav></nav>: representa una sección de una página que enlaza a otras páginas o a otras partes dentro de la página. No todos los grupos de enlaces en una página necesita estar en un elemento <nav>, sólo las secciones que constan de bloques de navegación principales son apropiados para el elemento de navegación.
  • <footer></footer>: representa el encabezado de una sección. El elemento se utiliza para agrupar un conjunto de elementos h1-h6 cuando el título tiene varios niveles, tales como subtítulos o títulos alternativos.
  • <time></time>: representa o bien una hora (en formato de 24 horas), o una fecha precisa en el calendario gregoriano (en formato ISO), opcionalmente con un tiempo y un desplazamiento de zona horaria.

Estructura de un documento HTML

Como se ha mencionado anteriormente, las nuevas etiquetas semánticas introducidas en HTML5 aportan un significado concreto al documento que estamos definiendo y por lo tanto, afectan de una manera directa a la forma en la que se estructura el contenido. A continuación se mostrará cómo se puede convertir una página escrita en HTML4 con las nuevas etiquetas introducidas en HTML5

Estructura tradicional de un documento en HTML4

El siguiente código muestra una estructura "clásica" de un documento HTML, donde los diferentes contenidos de la web se encuentran agrupados por etiquetas <div>. Por ellas solas, estas no aportan ningún tipo de significado, y el atributo id tampoco se le proporciona. Por ejemplo, si cambiamos <div id="header"> por <div id="cualquiercosa">, el significado sigue siendo el mismo, ninguno.

                        "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

                    </head>
                    <body>
                        <div id="header">
                            <a href="/"><img src=logo.png alt="home"></a>

                                Programación Web
                            </p>

                        <div id="nav">

                                    <a href="#" title="link to this post">Linky</a>
