Configuración básica de Git

Guía de apoyo| Miercoles 29 de agosto de 2018.

Introdución.

Esta guía está destinada a facilitar el uso de la herramienta de control de versiones Git, en conjunto con el servicio en línea de Github.com, sin embargo se considera válida para otros servicios git en la web.

Guía de configuración.

En primer lugar debemos crear un repositorio en cualquier servicio de control de versiones. En este caso se hará uso del siguiente repositorio ficticio: https://github.com/DouglasHdezT/repositorio.git; además se deberá establecer un espacio de trabajo local (crear un directorio), con el fin de mantener un orden al momento de trabajar. En este caso será el directorio "/home/douglashdezt/Escritorio/PW-2018"

Luego de esto, instanciamos y configuramos nuestro repositorio local, junto a su conexión remota a github. Para ello se debe iniciar la terminal en la carpeta de nuestro espacio de trabajo y se procede a ingresar los siguientes comandos:

git init git remote add <name_remote> <link_repository>

En nuestro caso particular aplicaría de la siguiente forma:

git init git remote add origin https://github.com/DouglasHdezT/repositorio.git

En el caso que nos encontremos en las maquinas del laboratorio, las cuales no tienen configuradas las variables globales de git (Usuario y correo electrónico), habrá que configurarlas de forma local al repositorio, ingresando en la terminal:

git config user.name "username" git config user.email "email@service.com"

En caso se quieran configurar de manera global en una maquina personal, solo se debe agregar el atributo --global a las sentencias anterior. De esta forma solo se configurá una sola vez, pero las credenciales quedan registradas en la máquina.

Luego de esta configuración se podrá proseguir a crear y editar archivos, directorios, etc. Cuando se quiera registrar un cambio y guardarlo en el servicio de github, se deberan ingresar los siguientes comandos:

git add <directorio/archivo> git commit -m "<message>" git push <remote> <branch> En el caso particular en el que se está trabajando, será de la siguiente forma: git add .
// Esta instrucción añade todos los archivos modificados en el directorio git commit -m "initial commit" git push origin master

Si todo se ha realizado bien, en este instante la consola padirá las credenciales del usuario configurado. Se debe tomar en cuenta que estos últimos tres comandos deben de realizarse cada vez que se quiera actualizar el repositorio en línea.
